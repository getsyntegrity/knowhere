# 认证和 API Keys

了解如何在 Knowhere AI 中管理认证和 API Keys。

## 认证方式

Knowhere AI 支持多种认证方式：

### 1. OAuth 登录
- **Google**：使用 Google 账户登录
- **GitHub**：使用 GitHub 账户登录  
- **Apple**：使用 Apple ID 登录

### 2. 邮箱密码登录
- 使用邮箱和密码直接登录
- 支持邮箱验证

## API Key 管理

### 创建 API Key

1. 登录 [控制台](https://knowhere.ai/dashboard)
2. 进入 "API Keys" 页面
3. 点击 "创建新的 API Key"
4. 输入 Key 名称（可选）
5. 点击 "生成 Key"
6. **重要**：立即复制并保存您的 API Key，它只会显示一次

### API Key 格式

API Key 格式为：`kwh_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx`

### 使用 API Key

在 HTTP 请求头中包含 API Key：

```bash
curl -H "Authorization: Bearer kwh_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx" \
     https://api.knowhere.ai/v1/table-fill/jobs
```

或在 SDK 中配置：

```python
from knowhere_sdk import KnowhereClient

client = KnowhereClient(api_key="kwh_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx")
```

```typescript
import { KnowhereClient } from '@knowhere/sdk-typescript';

const client = new KnowhereClient({
  apiKey: 'kwh_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx'
});
```

### 管理 API Key

#### 查看所有 API Keys
在控制台的 "API Keys" 页面查看所有已创建的 Keys。

#### 撤销 API Key
1. 在 API Keys 列表中找到要撤销的 Key
2. 点击 "撤销" 按钮
3. 确认撤销操作

**注意**：撤销后的 API Key 将立即失效，无法恢复。

#### 重命名 API Key
1. 点击 API Key 旁边的编辑图标
2. 输入新的名称
3. 保存更改

## 安全最佳实践

### 1. 保护您的 API Key
- **永远不要**在客户端代码中硬编码 API Key
- **永远不要**将 API Key 提交到版本控制系统
- 使用环境变量存储 API Key

```bash
# 设置环境变量
export KNOWHERE_API_KEY="kwh_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"
```

```python
import os
from knowhere_sdk import KnowhereClient

api_key = os.getenv('KNOWHERE_API_KEY')
client = KnowhereClient(api_key=api_key)
```

### 2. 定期轮换 API Key
- 定期创建新的 API Key
- 撤销不再使用的旧 Key
- 更新所有使用旧 Key 的应用程序

### 3. 监控 API Key 使用
- 定期检查 API Key 的使用情况
- 监控异常的使用模式
- 及时撤销可疑的 Key

## 错误处理

### 常见错误码

| 错误码 | 说明 | 解决方案 |
|--------|------|----------|
| 401 | 未授权 | 检查 API Key 是否正确 |
| 403 | 禁止访问 | 检查 API Key 是否有效 |
| 429 | 请求过多 | 降低请求频率 |

### 错误响应示例

```json
{
  "error": {
    "code": "unauthorized",
    "message": "Invalid API key",
    "details": "The provided API key is invalid or has been revoked"
  }
}
```

## 故障排除

### API Key 不工作
1. 确认 API Key 格式正确
2. 检查 API Key 是否已撤销
3. 确认请求头格式正确
4. 检查网络连接

### 认证失败
1. 确认使用的是正确的 API Key
2. 检查 API Key 是否过期
3. 确认账户状态正常
4. 联系技术支持
